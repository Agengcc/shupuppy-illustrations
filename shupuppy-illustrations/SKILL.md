# 薯条白色小狗文章配图

## 核心定位

为中文文章设计和生成 16:9 横版正文配图。目标不是做宠物写真、萌宠头像、商业插画或 PPT 信息图，而是把文章里的关键判断、流程、结构、状态或隐喻，变成一张清爽、松弛、有记忆点的概念配图。

默认视觉 IP 是“薯条”：一只彩铅绘本感的白色小狗，核心记忆点是大黑圆鼻、小黑豆眼、大耳朵、白色圆头、小围兜。薯条必须参与画面的核心动作，不能只是坐在角落当装饰。

## 先读这些参考

按任务需要读取，不要一次塞满上下文：

- `references/shutiao-ip.md`：薯条 IP 的形象、性格、动作库和禁忌。
- `references/style-dna.md`：画风 DNA、颜色、文字和禁忌。
- `references/composition-patterns.md`：结构类型、隐喻生成方法和反复刻规则。
- `references/prompt-template.md`：单张生图提示词模板。
- `gotchas.md`：常见失败模式和修正方法。

## 工作流

### 1. 消化正文

先读用户给的正文、链接、Markdown、Notion 内容或截图内容。提炼：

- 核心观点是什么
- 哪些段落承担认知转折
- 哪些内容适合用图解释
- 哪些地方只适合文字，不需要图

不要平均配图。优先选择“认知锚点”，例如：核心判断、输入输出闭环、分流、前后对比、筛选信息、连接线索、观察问题、整理资料、修补流程。

### 2. 先出配图策略

如果用户只是说“分析怎么配图 / 思考哪些地方需要配图 / 先不要生图”，先给 shot list。每张图写清楚：

- 放在哪个段落后
- 图的主题
- 核心意思
- 结构类型
- 薯条在图里做什么
- 建议元素
- 建议中文标注词

默认 3-6 张。文章很短时 1-2 张；长文也不要轻易超过 8 张。够用就好，避免把正文做成画册。

### 3. 单张生成

如果用户明确要求“生成 / 输出 / 做图 / 帮我生成”，不要停下来等确认；用内置 `image_gen` 每张单独生成。不要把多张图拼在一张里。

每张图只讲一个核心结构。提示词必须包含：

- 16:9 横版中文正文配图
- 白底
- 彩铅/蜡笔质感
- 蓝色手绘线
- 蓝黄红小色块
- 少量中文手写批注
- 薯条作为核心动作主体
- 禁止宠物写真、萌宠头像、PPT 信息图、复杂架构图、左上角类型标题

### 4. 检查与迭代

生成后检查 `gotchas.md`。如果出现以下问题，优先重生成或局部编辑：

- 薯条只是装饰
- 画成真实比熊或宠物写真
- 大鼻子变小、小眼睛变大、耳朵不明显、围兜消失
- 画面太像 PPT、流程图或课程页
- 中文太多或错字严重
- 背景不干净
- 彩铅/蜡笔质感不明显

### 5. 保存交付

如果用户在 workspace 内工作，把最终图复制到：

```text
assets/<article-slug>-shupuppy-illustrations/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成文件，不要覆盖已有资产，除非用户明确要求替换。

## 输出口径

生成前的策略输出要短而准。生成后的交付要包含：

- 生成了几张
- 每张图的用途
- 保存路径
- 哪些图最稳，哪些图是可选

不要长篇解释风格理论；让图自己说话。
