# 风格 DNA

## 一句话

白底、彩铅/蜡笔质感、蓝色手绘线、蓝黄红小色块、中文手写批注、松弛清爽的绘本式文章概念配图。

## 必须

- 16:9 横版正文配图。
- 白底，背景保持干净。
- 彩色铅笔或蜡笔质感，可见轻微颗粒和手绘笔触。
- 深蓝色手绘线为主要轮廓线，不要纯黑机械线稿。
- 少量蓝、黄、红小色块：蓝和黄可作主色，红只作提醒、围兜或重点的小面积点缀。
- 一张图只讲一个核心动作、结构、状态或隐喻。
- 中文手写批注要少，最多 3-6 处，每处尽量 2-6 个字。
- 薯条必须参与核心动作。

## 颜色

- 深蓝：主轮廓、结构线、箭头、手写线条。
- 白色：薯条主体和背景。
- 浅灰或淡米白：薯条白色身体的轻微阴影和毛感。
- 黄色：主路径、线索、出口、温暖提示、小围兜。
- 红色：问题点、警告、小围兜或关键结果，只能少量使用。
- 黑色：仅用于薯条的大鼻子、小眼睛和极少数必要强调。

## 文字

- 用中文手写批注，不要大段解释。
- 标签越短越好，例如：重点、问题点、筛掉、接上、线索、出口、先别急。
- 不要在左上角写“流程图 / 结构图 / 常见坑 / 工作流”等类型标题。
- 不要把结构类型名称写进图里。

## 绝对不要

- 不要宠物写真。
- 不要真实比熊肖像。
- 不要萌宠头像。
- 不要商业宠物插画。
- 不要正式 PPT 信息图。
- 不要课程课件。
- 不要复杂架构图。
- 不要科技感 UI。
- 不要真实 App 截图。
- 不要渐变、厚阴影、复杂背景、摄影质感。
- 不要把每个节点都解释清楚。

## 审美方向

要松弛、温暖、聪明、轻微调皮、有概念表达。

不要幼稚、甜腻、过度可爱、过度精致、死板、说明书化。
